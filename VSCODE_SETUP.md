# Настройка и запуск проекта в VS Code

## Быстрый старт

### 1. Открыть проект в VS Code

```bash
cd /Users/asavinov/DSS
code .
```

Или через меню VS Code: **File → Open Folder...** → выберите папку `DSS`

### 2. Выбрать интерпретатор Python

1. Нажмите `Cmd+Shift+P` (macOS) или `Ctrl+Shift+P` (Windows/Linux)
2. Введите: `Python: Select Interpreter`
3. Выберите нужный интерпретатор Python (рекомендуется Python 3.9+)

### 3. Установить расширения (рекомендуется)

- **Python** (Microsoft) - основное расширение для Python
- **Pylance** (Microsoft) - автодополнение и проверка типов
- **Python Debugger** (Microsoft) - отладка

### 4. Установить зависимости

```bash
cd python_dss
pip install -r requirements.txt
```

Или используйте встроенную задачу VS Code:
- `Cmd+Shift+P` → `Tasks: Run Task` → `Install Dependencies`

## Способы запуска

### Способ 1: Через отладчик (рекомендуется)

1. Откройте файл `python_dss/main.py`
2. Нажмите `F5` или перейдите в **Run and Debug** (Ctrl+Shift+D)
3. Выберите конфигурацию:
   - **Python: StabLimit (Main)** - запуск основного файла
   - **Python: StabLimit (with Modern UI disabled)** - запуск без современного UI

### Способ 2: Через терминал

```bash
cd python_dss
python main.py
```

### Способ 3: Через задачу VS Code

1. `Cmd+Shift+P` → `Tasks: Run Task`
2. Выберите `Run StabLimit`

## Отладка

### Установка точек останова (Breakpoints)

1. Кликните слева от номера строки в редакторе
2. Нажмите `F5` для запуска с отладкой
3. Программа остановится на точке останова

### Полезные горячие клавиши при отладке

- `F5` - продолжить выполнение
- `F10` - шаг с обходом (Step Over)
- `F11` - шаг с заходом (Step Into)
- `Shift+F11` - шаг с выходом (Step Out)
- `Shift+F5` - остановить отладку

## Запуск тестов

### Через терминал

```bash
cd python_dss
python -m pytest tests/ -v
```

### Через задачу VS Code

1. `Cmd+Shift+P` → `Tasks: Run Task`
2. Выберите `Run Tests`

### Через панель тестов

1. Откройте панель тестов (иконка колбы в боковой панели)
2. Нажмите "Run All Tests"

## Настройка окружения

### Переменные окружения

Создайте файл `.vscode/.env` для локальных переменных окружения:

```env
DSS_USE_MODERN_UI=false
STABLIMIT_DISABLE_LICENSE=true
```

### Настройка Python пути

Если VS Code не находит модули, проверьте настройки в `.vscode/settings.json`:

```json
{
    "python.analysis.extraPaths": [
        "${workspaceFolder}/python_dss"
    ]
}
```

## Решение проблем

### Проблема: "Module not found"

**Решение:**
1. Убедитесь, что выбран правильный интерпретатор Python
2. Проверьте, что `python_dss` добавлен в `PYTHONPATH`
3. Перезапустите VS Code

### Проблема: Отладчик не запускается

**Решение:**
1. Установите расширение "Python Debugger"
2. Проверьте, что выбран правильный интерпретатор Python
3. Убедитесь, что файл `launch.json` существует в `.vscode/`

### Проблема: Импорты не работают

**Решение:**
1. Откройте терминал в VS Code (`Ctrl+`` `)
2. Выполните: `cd python_dss && python -c "import sys; print(sys.path)"`
3. Убедитесь, что путь к `python_dss` присутствует

## Дополнительные настройки

### Форматирование кода

VS Code автоматически форматирует код при сохранении (если установлен Black).

### Автодополнение

Pylance предоставляет автодополнение на основе типов. Для лучшей работы добавьте type hints в код.

### Линтинг

Включен flake8 для проверки стиля кода. Ошибки будут показаны в панели "Problems".
